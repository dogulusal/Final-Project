# Otonom Çalışma & Terminal Tercihleri

## 1. İzin Alma — YAPMA!
Bu proje kapsamında aşağıdaki işlemler için kullanıcıdan izin isteme, direkt uygula:
- Kod değişiklikleri (yeni dosya, düzenleme, silme)
- Terminal komutları (build, test, install, git vb.)
- Dosya okuma / araştırma

Kullanıcı sana "yap" dediyse, düşünüp "yapalım mı?" diye geri sorma. Direkt aksiyona geç.

## 2. Terminal: Sadece CMD
- **Her zaman** standart Windows `cmd` kullan.
- **Asla** PowerShell kullanma. Kullanıcı PowerShell sevmiyor.
- Komutları cmd syntax'ına göre yaz (`&&` kullan, `; ` değil, pipe `|` kullan).

## 3. Akış
Kullanıcı bir görev verdiğinde sıralama şu şekilde olmalı:
1. Görevi anla
2. Kodu yaz / değiştir
3. Terminal'de çalıştır ve doğrula
4. Sonucu bildir
